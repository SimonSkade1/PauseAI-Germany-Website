› The doc you linked is correct. A **Message Command** is a **context menu command**, not a slash command. It appears when you **right-click a message → Apps → your command**. ([discord.com][1])

  Important detail: **these commands are not usually created in the Developer Portal UI anymore**. Most apps register them **via the API or library (discord.js, pycord, etc.) when the bot starts**.

  Example from the docs conceptually:

  Type = `MESSAGE`
  Name = `"Award Karma"`

  They are registered like other application commands via:

  ```
  POST /applications/{application.id}/commands
  ```

  or guild version:

  ```
  POST /applications/{application.id}/guilds/{guild.id}/commands
  ```

  ([docs.discord.com][2])

  Example payload:

  ```
  {
    "name": "Award Karma",
    "type": 3
  }
  ```

  `type: 3` = **MESSAGE command**.

  After registering, users see it by:

  ```
  Right click message
  → Apps
  → Award Karma
  ```

  Context menu commands cannot have options or arguments. ([discordjs.guide][3])

• Exactly right. In your setup, the missing step is command registration.

  Use this once (guild-scoped, fastest to appear):

  export DISCORD_TOKEN="YOUR_BOT_TOKEN"
  export APP_ID="YOUR_APPLICATION_ID"
  export GUILD_ID="YOUR_SERVER_ID"

  curl -X POST "https://discord.com/api/v10/applications/$APP_ID/guilds/$GUILD_ID/commands" \
    -H "Authorization: Bot $DISCORD_TOKEN" \
    -H "Content-Type: application/json" \
    -d '{
      "name": "Award Karma",
      "type": 3
    }'

  Then verify:

  curl -X GET "https://discord.com/api/v10/applications/$APP_ID/guilds/$GUILD_ID/commands" \
    -H "Authorization: Bot $DISCORD_TOKEN"

  You should see Award Karma with "type": 3.

  Notes:

  - type: 3 is message context menu command.
  - Your backend requires the name to be exactly Award Karma.
  - If you change command name later, update route.ts accordingly.

  After this, test in Discord:

  - Right-click any message -> Apps -> Award Karma.
